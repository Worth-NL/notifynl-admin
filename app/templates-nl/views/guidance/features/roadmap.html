Routekaart

{{ content_metadata( data={ "Laatst bijgewerkt": "20 maart 2025", "Volgende beoordeling": "20 juni 2025" } ) }}

De routekaart van NotifyNL laat zien waar we aan werken en enkele zaken die we hebben afgerond.

De routekaart is slechts een richtlijn. Het omvat niet alles wat we doen, en sommige zaken kunnen veranderen.

U kunt contact met ons opnemen als u vragen heeft over de routekaart, wilt meedoen aan gebruikersonderzoek of suggesties heeft voor nieuwe functionaliteiten.

Waar we nu aan werken

  • Verbeteren van de manier waarop we onze prestaties meten
  • ‘Economy mail’ toevoegen aan de lijst met verzendopties
  • Het makkelijker maken om Notify te gaan gebruiken
  • Gebruikers laten deelnemen aan een bestaande dienst
  • Onze front-end code verbeteren om te voldoen aan WCAG 2.2 toegankelijkheidsdoelen
  • Websitegebruikers bestanden per e-mail laten versturen
  • Verbeteren van de Notify API
  • High-volume diensten een rapport laten downloaden van de berichten die zij hebben verstuurd
  • Onze beveiliging herzien volgens het Cyber Assessment Framework
  • Manieren onderzoeken om diensten te helpen betere berichten te schrijven
  • Onze interne monitoring- en waarschuwingssystemen bijwerken
  • Ons proces voor het opsporen van sms-fraude herzien

Waar we later aan werken

  • Nieuwe manieren onderzoeken om onze capaciteit te vergroten
  • Het begrijpelijker maken hoe sms-prijzen werken
  • Het facturatie- en betalingsproces verbeteren
  • Het makkelijker maken voor diensten om de gebruiksgegevens te zien die ze nodig hebben
  • Diensten via de API laten toevoegen en bewerken van berichtsjablonen
  • Manieren onderzoeken om de relatie tussen organisaties en hun diensten te vereenvoudigen
  • Meer organisaties hun eigen diensten laten goedkeuren (publieke bèta)

Wat we hebben gedaan

  • API callbacks toegevoegd voor geretourneerde brieven
  • Een nieuwe ontwikkelomgeving toegevoegd voor het prototypen en testen van nieuwe functionaliteiten
  • Onze jaarlijkse IT-gezondheidscheck uitgevoerd
  • Een nieuwe omgeving en deployment pipeline toegevoegd om ontwikkeling veiliger te maken
  • NCSC-richtlijnen voor NHS-diensten nageleefd door gebruik van een consistente sms-afzender-ID
  • Onze ontwerpcomponenten gemigreerd naar versie 5 van het GOV.UK Design System
  • Ontvangers kunnen zich met één klik afmelden voor e-mails
  • Diensten kunnen afmeldverzoeken beheren
  • JPG en PNG toegevoegd aan de lijst met bestandstypen die diensten per e-mail kunnen versturen
  • Moeilijker gemaakt om brieven naar niet-bestaande adressen te versturen
  • Nieuwe manieren gevonden om ontvangers te beschermen tegen sms-fraude
  • Diensten die de Notify API gebruiken kunnen hun facturatiegegevens inzien
  • Een update van onze gebruikersvoorwaarden aangekondigd
  • Verschillende problemen opgelost die vermeld staan in onze toegankelijkheidsverklaring
  • API-verbeteringen geïdentificeerd en geprioriteerd (ontdekking)
  • Onze applicaties gemigreerd van GOV.UK PaaS naar Amazon Elastic Container Service
  • Nieuwe notificatiekanalen onderzocht (ontdekking)
